# UI 介面佈局規格 (UI Layout)

## 🗣️ 內部討論紀錄 (Internal Discussion Record)

**任務：** Phase 1 - Task 7：設計 UI 介面佈局規格

**與會者：**
- 🔥 **Diablo 代表** — 敘事深度、黑暗氛圍、裝備驅動
- ⚡ **PoE 代表** — 系統複雜度、build 可能性
- 🛡️ **Arknights 代表** — 策略性塔位、角色魅力
- 🏰 **Kingdom Rush 代表** — 塔防核心樂趣、平衡性

---

**討論摘要：**

**Diablo 代表：** 「UI 需要简洁明了，让玩家专注于战斗。」

**PoE 代表:** 「复杂的装备/宝石系统需要清晰的 UI 来组织。」

**Arknights 代表：** 「参考 AK 的做法，UI 需要层次分明。」

**Kingdom Rush 代表：** 「TD 部分的 UI 需要适配手柄操作。」

---

**共識結論：**
- **风格：** 暗色主题 + 霓虹点缀
- **原则：** 战斗中最小化 UI，城镇中功能齐全
- **适配：** 键鼠 + 手柄双支持

## 1. 遊戲畫面配置 (In-Game HUD)

### 1.1 整体布局

```
┌─────────────────────────────────────────┐
│  [小地图]              [任务目标]      │
│                                         │
│                                         │
│           [游戏主视角]                   │
│                                         │
│                                         │
│  [干员栏]              [技能栏]        │
│  [部署点]              [统帅值]        │
│  [英雄血量]            [基地血量]      │
└─────────────────────────────────────────┘
```

### 1.2 区域说明

| 区域 | 内容 | 优先级 |
|------|------|--------|
| 左上 | 小地图 + 当前地图名 | 中 |
| 右上 | 任务目标/波次信息 | 中 |
| 底部左侧 | 干员部署栏 + 可用点数 | 高 |
| 底部右侧 | 技能栏 + 大招能量 | 高 |
| 底部中间 | 英雄血量 + 基地血量 | 高 |
| 中央 | 战斗主视角 | 最高 |

### 1.3 战斗时 UI 隐藏

- 战斗时隐藏非必要 UI
- 只保留：血量、部署点、技能栏
- 按 Tab 可暂时显示完整 UI

## 5. 通用 UI 元件

### 5.1 按钮状态

| 状态 | 样式 |
|------|------|
| 默认 | 暗色背景 + 亮边框 |
| 悬停 | 边框发光 + 轻微放大 |
| 按下 | 颜色变暗 |
| 禁用 | 灰色 + 半透明 |

### 5.2 颜色主题

- **主色：** 深灰 (#1a1a2e)
- **强调色：** 橙色 (#ff6b35) / 蓝色 (#4ecdc4)
- **稀有度颜色：**
  - 普通：白色 (#ffffff)
  - 精良：蓝色 (#4a90e2)
  - 传奇：金色 (#ffd700)

### 5.3 字体

- **标题：** Bold, 24px
- **正文：** Regular, 16px
- **数值：** Monospace, 18px
